# 엄마랑 후쿠오카 2박 3일 — 여행 스크랩북 PWA

## 1. GitHub Pages로 올리기 (아이패드/아이폰, 터미널 없이)

**GitHub 앱에는 "파일 업로드" 기능이 없어요.** 반드시 **Safari**에서 진행해야 해요.

1. 이미 `fukuoka-trip` 저장소를 만들었다면 그대로 사용
2. **Safari** 앱을 열고 `github.com` 접속 → 로그인
3. 저장소(`fukuoka-trip`) 페이지로 이동
4. 화면 우측 하단(또는 주소창 옆) **"aA" 아이콘 → "데스크탑 웹 사이트 요청"** 탭
   (이게 안 보이면 새로고침 버튼을 길게 눌러도 나와요)
5. 데스크탑 화면으로 바뀌면 **Add file → Upload files** 버튼이 보여요
6. 압축을 푼 폴더 안의 파일 9개를 **전부 선택**해서 업로드
   (`index.html, style.css, db.js, data.js, app.js, manifest.json, sw.js, icon-192.png, icon-512.png`
   — 폴더 없이 전부 같은 위치에 있어야 하니 이대로 올리면 됨)
7. 하단 **Commit changes**
8. 좌측 메뉴 **Settings → Pages** 이동 (여기도 데스크탑 모드 유지된 채로)
9. Branch를 **main / (root)** 로 설정 → Save
10. 1~2분 후 상단에 뜨는 `https://아이디.github.io/fukuoka-trip/` 링크로 접속

이 링크를 엄마/동행자에게 공유하면 각자 폰에서 접속 후
**"홈 화면에 추가"** 하면 앱처럼 아이콘이 생기고 오프라인에서도 열려요.

> Public repo라 링크를 아는 사람은 볼 수 있어요. 완전 비공개가 필요하면
> GitHub Pro(유료)의 private Pages 기능을 쓰거나, Netlify Drop 같은 서비스로
> 바꿔도 구조는 동일하게 그대로 씁니다.

## 2. 앱 사용법

- **일정 탭**: DAY1/2/3 탭 전환 → 각 카드 탭하면 메모 작성 + 사진/PDF 첨부 가능
- **지도 탭**: 구글 마이맵에서 만든 지도를 "공유 → 웹에 게시"로 받은 embed 링크를 등록하면
  앱 안에서 바로 전체 동선 지도가 보여요
- **예약함 탭**: 항공편/숙소/참고정보(공항↔역 이동, 교통카드, 환전메모)에
  캡처 이미지·PDF 예약확인서를 첨부해두는 곳
- **체크리스트 탭**: 여권, 트래블월렛 등 준비물 체크

모든 메모/사진/PDF/체크리스트는 **그 폰의 브라우저 저장소(IndexedDB)** 에만 저장돼요.
서버로 전송되지 않고, 앱을 지우거나 브라우저 데이터를 지우면 함께 사라지니
중요한 예약 확인서는 원본을 따로도 보관해두는 걸 추천해요.

## 3. 일정 내용 수정하고 싶을 때

`data.js` 파일 안 `ITEMS` 배열을 직접 수정하면 돼요. 각 항목 구조:

```js
{ id: "d1-4", day: 1, time: "15:30", tag: "food",
  title: "카페 · 신발 벗고 편안하게",
  desc: "일본 감성 가득한 다다미 카페에서 여유 타임",
  mapQuery: "하카타 카페" }
```

- `tag`: normal(파랑) / food(핑크) / onsen(초록) / shop(진남색) — 카드 위 워시테이프 색이 바뀜
- `mapQuery`: 모달 안 "구글 지도에서 위치 보기" 링크에 쓰일 검색어

수정 후 GitHub repo에 다시 `data.js` 파일만 골라 덮어쓰기 업로드하면 반영돼요
(Add file → Upload files로 같은 이름 파일을 올리면 자동으로 덮어써져요).
